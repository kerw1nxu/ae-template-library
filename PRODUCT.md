# PRODUCT

register: product

## Product Purpose

This project is an AE template library for collecting, searching, previewing, downloading, uploading, scanning, and organizing After Effects template assets. The interface should feel like a professional internal material library: efficient, trustworthy, media-first, and calm enough for repeated daily use.

## Users

- Visitors can only view the public homepage and the login page. They should understand what the library is, but they must not open template detail pages, preview protected media, download files, upload assets, scan folders, manage tags, or enter admin tools.
- Signed-in users can browse templates, search and filter, open detail pages, preview videos, and download template packages.
- Admin users can upload templates, scan the configured import folder, edit template tags, create and manage grouped tags, manage users, and disable or restore accounts.

## Strategic Principles

- The template media is the product. Covers and preview videos should carry the visual weight before decorative UI.
- Keep the permission boundary visible but not noisy: anonymous users get a clear login path; authenticated users get working-library controls; admins get management actions where they are needed.
- Favor fast scanning, clear density, and stable layouts over marketing decoration.
- Chinese interface copy should be direct, concise, and free of encoding artifacts.
- Do not widen anonymous access during UI work unless the security boundary is explicitly changed.

## Tone

Professional, clear, and restrained. Avoid slogan-heavy copy, exaggerated enthusiasm, and generic SaaS language. Use practical labels that describe the actual workflow.

## Anti-References

- Generic landing pages with large ornamental hero graphics.
- Purple or blue AI-style gradients used as the main identity.
- Nested cards, oversized rounded rectangles, and decorative glass effects.
- Interfaces where admin actions compete visually with normal browsing and download tasks.
